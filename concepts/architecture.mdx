## Technology Choices

### Frontend

<AccordionGroup>
  <Accordion title="Next.js 14 (App Router)">
    Server components reduce client bundle size. Built-in API routes for prototyping. Excellent Vercel integration and a huge ecosystem. App Router enables layouts, streaming, and server-side data fetching without extra setup.
  </Accordion>
  <Accordion title="Tailwind CSS + shadcn/ui">
    shadcn/ui is not a component library вҖ” it's copy-paste components built on Radix + Tailwind. No dependency bloat, full customization. You own the components and can modify them freely.
  </Accordion>
</AccordionGroup>

### Backend

<AccordionGroup>
  <Accordion title="Cloudflare Workers + Hono.js">
    Hono is a lightweight, fast framework built for edge runtimes. It has native D1/R2/Queue bindings. Workers handle 100K requests/day on the free tier and deploy globally with zero config.
  </Accordion>
  <Accordion title="Cloudflare D1">
    SQLite-based relational database. Perfect for the audit-heavy, relational data model. The free tier (5M reads/day, 100K writes/day) is generous for early-stage products.
  </Accordion>
  <Accordion title="Cloudflare R2">
    S3-compatible object storage for dataset JSONL files and eval report exports. 10 GB free storage and 10M reads/month.
  </Accordion>
  <Accordion title="GitHub OAuth">
    Free, all target users have GitHub accounts, and simple to implement. No third-party auth service dependency.
  </Accordion>
</AccordionGroup>
